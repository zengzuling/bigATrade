# A股一周强势股捕捉系统设计

## 目标

第一版做一个命令行工具，用免费 AkShare 数据筛选 A 股强势股，生成一周交易计划，并回测未来 5 个交易日是否达到 10% 以上涨幅。

核心目标不是保证推荐必涨，而是把推荐逻辑变成可验证的数据闭环：

1. 从 A 股全市场拉取历史行情。
2. 用规则筛出强势候选股。
3. 生成买入区间、目标价、止损价、持有周期和推荐原因。
4. 回测历史推荐在未来 5 个交易日内是否触发 10% 目标。
5. 输出 CSV / JSON 报告，便于人工复核。

## 非目标

第一版不做自动下单，不接 easytrader，不接 qteasy，不接实时盘口，不接新闻资讯，不做实盘交易。

这些能力放到后续阶段，前提是第一版回测结果能证明策略有继续投入价值。

## 技术选择

### 主体框架

使用 Python 自写轻量框架。

原因：当前目标是 5 个交易日内冲击 10% 的强势股策略，规则明确，回测周期短。直接用 pandas 写指标和回测更短、更透明，也更容易针对 A 股涨停、停牌、ST 等规则做定制。

### 数据源

第一版使用 AkShare 免费数据。

需要的数据：

- A 股股票列表。
- 日线行情：开盘价、最高价、最低价、收盘价、成交量、成交额。
- 股票名称，用于排除 ST、退市风险股票。

### 依赖

推荐依赖：

- `akshare`
- `pandas`
- `numpy`
- `typer`
- `pytest`

## 命令行接口

第一版提供两个核心命令：

```powershell
python -m bigatrade recommend --date 2026-06-05 --top 30
python -m bigatrade backtest --start 2025-01-01 --end 2026-06-05
```

### recommend

按指定交易日生成推荐列表。

输出字段：

- 推荐日期
- 股票代码
- 股票名称
- 收盘价
- 推荐买入价下限
- 推荐买入价上限
- 目标价
- 止损价
- 目标涨幅
- 最大持有交易日
- 强势评分
- 推荐原因
- 风险提示

### backtest

对历史日期逐日生成推荐并验证未来 5 个交易日表现。

输出字段：

- 推荐日期
- 股票代码
- 股票名称
- 买入价
- 目标价
- 止损价
- 实际买入日期
- 实际卖出日期
- 未来 5 日最高涨幅
- 是否达到 10%
- 实际收益率
- 退出原因：止盈 / 止损 / 到期 / 未触发买入

## 筛选规则

第一版使用强势股规则，不做基本面价值投资。

候选股票必须满足：

1. 非 ST、非退市风险。
2. 最近 30 个交易日数据完整。
3. 最近 20 日成交额不能过低，避免流动性不足。
4. 当前收盘价高于 5 日、10 日、20 日均线。
5. 最近 5 日涨幅为正，但不过度透支。
6. 最近 1-3 日成交量高于过去 10 日均量。
7. 当前价格接近或突破最近 20 日新高。

强势评分由以下部分组成：

- 趋势分：价格站上多条均线。
- 量能分：近期成交量放大。
- 突破分：接近或突破 20 日高点。
- 动量分：最近 5 日和 10 日涨幅。
- 风险扣分：短期涨幅过大、长上影、放量长阴。

## 交易计划规则

买卖价格由规则生成，不由 AI 直接猜。

第一版规则：

- 推荐买入价下限：推荐日收盘价下方 1%。
- 推荐买入价上限：推荐日收盘价上方 2%。
- 目标价：买入中位价乘以 1.10。
- 止损价：买入中位价下方 5%。
- 最大持有周期：5 个交易日。

后续可以把目标价改成压力位、ATR、涨停空间等更复杂规则。

## 回测规则

回测从推荐日后的下一个交易日开始。

买入规则：

- 如果未来 5 个交易日内最低价小于等于买入价上限，并且最高价大于等于买入价下限，认为触发买入。
- 成交价使用买入区间中位价。

卖出规则从实际买入日开始，按交易日顺序逐日扫描：

1. 如果某日最低价触发止损，按止损价卖出。
2. 如果某日最高价触发目标价，按目标价卖出。
3. 如果同一天同时触发止盈和止损，第一版按保守规则处理为止损。
4. 如果买入后持有满 5 个交易日仍未触发止盈或止损，按最后一个持有日收盘价卖出。
5. 如果 5 个交易日内未触发买入，记录为未触发买入。

## AI 层边界

第一版可以先不接大模型，直接由规则生成推荐原因。

后续接 AI 时，AI 只负责解释和归纳：

- 为什么入选。
- 有哪些风险。
- 适合什么交易纪律。
- 回测表现如何复盘。

AI 不直接决定买入价、目标价、止损价。

## 测试策略

第一版优先测试不依赖网络的核心逻辑：

1. 指标计算是否正确。
2. 强势评分是否能区分强势样本和弱势样本。
3. 交易计划价格是否符合 10% 目标和 5% 止损。
4. 回测引擎是否正确处理止盈、止损、到期、未触发买入。

AkShare 网络请求只做薄封装，先不强测真实接口。

## 验收标准

第一版完成时应满足：

1. 可以安装依赖并运行测试。
2. `recommend` 可以输出指定日期的候选股 CSV。
3. `backtest` 可以输出一段历史区间的回测明细和汇总。
4. 回测汇总包含推荐次数、触发买入次数、10% 达标率、平均收益率、最大单笔亏损。
5. 所有新增核心逻辑有自动化测试。
